Change Requests

If you do not have editing permissions on the site, then log in to your github account and submit a new issue, Check the current issues first in case someone else has reported it.

I'll be notified of new issues and will respond to them. You'll get a notification when the status changes.

Page Editing

If you do have permission to edit content, then the easiest way is via the github editor. Just go to the source site, and find the page you want to edit. Use the github text editor, commit the change with an optional comment.

The live site is not updated immediately, but should update within five minutes. If there is no update after 30 minutes, then submit a new issue with the details.

To add new content, there is a "+" sign on the github page for each folder. Create a new page with the correct header metadata for your content (you need "extends", "title" and "description" for a plain page, for example). Check existing content for examples. Again it takes a few minutes for changes to go through to the live site.

Mistakes

All the content is versioned and history is kept, so mistakes can be rolled back. You can submit a new issue if you can't do this yourself.
